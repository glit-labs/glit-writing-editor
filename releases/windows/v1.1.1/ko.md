## 개선 사항

### Windows: 타이틀바 색상을 앱 테마에 연동

Window Controls Overlay 방식으로 모든 창(메인·미리보기·마크다운·이미지·PDF)의 네이티브 타이틀바가 앱 테마 색상과 일치하도록 변경되었습니다.

- 테마 변경 시 시스템 버튼(최소화·최대화·닫기) 색상이 즉시 업데이트됩니다.
- 기존 Toolbar 컴포넌트가 타이틀바 역할을 겸합니다.
- macOS는 기존 동작(hiddenInset)을 유지합니다.

### 빈 화면 — [폴더 열기] / [웹소설 집필 시작] 버튼

편집기 영역이 비어 있을 때 표시되는 안내 텍스트가 두 개의 버튼으로 교체되었습니다.

- **[폴더 열기]** 버튼: 폴더를 선택해 워크스페이스로 열 수 있습니다.
- **[웹소설 집필 시작]** 버튼: 웹소설 집필 모드를 바로 시작합니다. 기존 워크스페이스 수에 따라 모달이 분기됩니다.
- 워크스페이스·웹소설 모드 활성화 상태에 따라 버튼 accent 색상이 변경됩니다.
- 8개 UI 언어 완전 지원.

### 웹소설 집필 모드 활성화 시 폴더 트리 자동 펼침

집필 모드를 켤 때 작품·집필 중·세계관·완성 원고 폴더가 자동으로 펼쳐져 핵심 구조를 별도 조작 없이 바로 확인할 수 있습니다.

## 버그 수정

### Windows: 이미지·PDF·마크다운 이미지 미표시

Windows에서 에디터에 삽입된 이미지, PDF 뷰어, 마크다운 미리보기 이미지가 표시되지 않는 문제가 수정되었습니다.

- `net.fetch('file://')` Windows 실패를 `fs.readFile` 기반으로 교체.
- Windows 드라이브 경로 정규화(리딩 슬래시 제거 및 콜론 복원).
- PDF 새 창 경로의 백슬래시 정규화.
- 마크다운 이미지 URI 인코딩 시 콜론 보존(`encodeURI` 적용).

### Windows: 자정 테마 타이틀바 버튼 호버 색상 미표시

자정 테마의 어두운 배경에서 Windows 타이틀바 버튼 호버 색상이 보이지 않던 문제가 수정되었습니다. `nativeTheme.themeSource`를 앱 테마에 연동하여 올바른 다크/라이트 모드 호버 색상이 적용됩니다.
