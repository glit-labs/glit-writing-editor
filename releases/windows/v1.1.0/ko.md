# Glit 릴리즈 노트 — v1.1.0

**출시일:** 2026-05-31  
**이전 버전:** v1.0.1

## 새로운 기능

### PDF 뷰어

PDF 파일을 에디터 내 인라인 또는 독립 창에서 열 수 있습니다.

- 어떤 배율에서도 페이지가 선명하게 표시됩니다.
- **−** / **+** 버튼 또는 `Cmd+스크롤` (macOS) / `Ctrl+스크롤` (Windows/Linux)로 확대·축소합니다. **⊡** 버튼을 누르면 현재 창 너비에 맞춰 페이지가 자동 조절됩니다.
- PDF 내 텍스트를 선택하고 복사할 수 있으며, 어떤 배율에서도 선택 영역이 정확하게 표시됩니다.
- **↗** 버튼으로 독립 창을 열 수 있으며, 파일명과 줌 컨트롤이 제공됩니다.
- 사이드바에서 PDF 파일에 확장자 배지가 표시되고, 클립보드 복사·붙여넣기도 지원됩니다.
- PDF 파일은 안전하게 열립니다. PDF 내 스크립트는 비활성화되고 외부 리소스 로딩은 차단됩니다.

### 마크다운 Front Matter — 파싱 및 시각화

`.md` 파일 상단의 YAML 메타데이터(Front Matter)를 파싱하여 시각화합니다.

- `---` 구분자로 감싼 Front Matter 블록을 인식하고 파싱합니다.
- Front Matter 블록은 미리보기, 독립 창, 보조 사이드바에서 숨겨지며, 본문만 표시됩니다.
- 에디터에서는 Front Matter 영역이 시각적으로 강조되고, 키·값이 현재 테마에 맞춰 색상으로 구분됩니다.
- 글꼴 크기 또는 줄간격 변경 시 강조 영역이 자동으로 업데이트됩니다.

### 마크다운 프리뷰 이미지 클릭 시 이미지 뷰어 창 열기

마크다운 미리보기의 이미지 위에 마우스를 올리면 **"↗ 새 창으로 보기"** 오버레이 버튼이 나타납니다. 버튼을 클릭하면 독립 이미지 뷰어 창에서 이미지를 확인할 수 있습니다.

### 텍스트 파일 확장자 무관 복사·붙여넣기 및 편집 지원

파일의 텍스트 여부를 확장자가 아닌 파일 내용으로 판별합니다. 확장자가 없거나 낯선 확장자의 파일도 바이너리 데이터가 없으면 에디터에서 편집하고 복사·붙여넣기할 수 있습니다.

### 워크스페이스 닫기 시 연관 창 함께 닫기

사이드바에서 워크스페이스의 **×** 버튼을 누르면, 해당 워크스페이스 내 파일로 열려 있는 이미지·마크다운 미리보기·PDF 독립 창이 함께 닫힙니다.

### 외부 드래그앤드롭: 폴더 및 지원 파일 전체 허용

이전에는 이미지 파일만 드래그하여 추가할 수 있었습니다. 이제 폴더와 기타 지원 파일 형식(`.txt`, `.md`, `.pdf` 등)도 드래그로 추가할 수 있으며, 클립보드 붙여넣기와 동일하게 동작합니다.

### 클립보드로 외부 파일 워크스페이스에 복사

워크스페이스 외부의 파일을 클립보드(`Cmd/Ctrl+C` → `Cmd/Ctrl+V`)로 워크스페이스에 복사할 수 있습니다. 이름 충돌 시 자동 숫자 접미사로 처리됩니다.

### 설정 — 언어 선택 드롭다운 테마 스타일 적용

설정의 언어 선택 UI가 4가지 테마에 자동으로 대응하는 커스텀 드롭다운으로 교체되었습니다. 외부 클릭 또는 Escape 키로 닫히며, 선택된 언어는 accent 색상으로 강조 표시됩니다.

### 휴지통

사이드바에서 삭제한 파일·폴더가 즉시 영구 삭제되지 않고 앱 데이터 내 휴지통으로 이동됩니다.

- 삭제된 항목은 앱 데이터 폴더의 전용 휴지통 폴더에 원본 위치·삭제 일시와 함께 보관됩니다.
- **설정 → 휴지통** 탭: 워크스페이스별로 카드 형태로 그룹화되며, 헤더는 워크스페이스 이름과 전체 경로를 두 줄로 분리 표시합니다. 삭제 일시와 폴더 내부 파일 목록 미리보기도 제공됩니다.
- 각 항목 이름 아래에 워크스페이스 내 상대 경로가 표시됩니다.
- 항목별 **복구** 버튼 및 **삭제** 버튼(영구 삭제). 삭제 전 확인 팝업이 표시됩니다.
- 복구 충돌 시 팝업 — **모두 표시** (`_복구` 접미사, 이미 존재하면 순번 추가) / **덮어쓰기** 선택
- 워크스페이스별 **비우기** 및 전체 **모두 비우기** 버튼, 각각 확인 팝업 후 실행. **모두 비우기** 버튼은 accent 색상으로 강조됩니다.
- 사이드바에서 삭제 즉시 휴지통 탭 목록 실시간 갱신
- 8개 UI 언어 완전 지원, 언어별 복구 접미사 적용

### 자동 백업

워크스페이스 안에서 텍스트 파일을 편집하는 동안 예기치 못한 오류·실수로부터 원고를 보호하기 위해 백업본이 자동으로 생성됩니다.

- **세션 원본**: 파일을 저장할 때마다 현재 내용이 워크스페이스 내 숨김 백업 폴더에 복사됩니다.
- **주기 스냅샷**: 설정한 주기로 타임스탬프가 포함된 스냅샷이 저장됩니다. 직전 스냅샷과 내용이 동일하면 건너뛰고, 최대 개수를 초과하면 가장 오래된 것부터 자동 삭제됩니다.
- **설정 → 자동 백업** 섹션: on/off 토글, 백업 주기(기본 5분), 최대 백업 개수(기본 10개), "오래된 백업 자동 정리" 옵션(기본 OFF) + 보관 기간(기본 30일)
- 워크스페이스 내 텍스트 파일만 백업됩니다. 바이너리 파일과 워크스페이스 외부 파일은 제외됩니다.
- 백업 폴더는 숨김 처리되어 사이드바에 표시되지 않습니다.
- 8개 UI 언어 완전 지원

### 미리보기(디바이스뷰): 다중 독립 인스턴스 지원

에피소드별 독립 미리보기 창을 여러 개 열 수 있습니다.

- 사이드바에서 에피소드 우클릭 시 **"미리보기 열기"** 메뉴 항목이 추가되었습니다. 다른 에피소드는 각자 독립 창으로 열리며, 같은 에피소드를 다시 클릭하면 기존 창이 포커스됩니다.
- 툴바의 **미리보기** 버튼과 **WnReview** 패널 미리보기 버튼은 진입 경로에 관계없이 파일별 동일 창을 공유합니다.
- WnReview 패널에 **미리보기** 버튼이 추가되었습니다.
- 기본 폰트 크기가 9px에서 16px로 변경되었습니다.
- 미리보기 창은 폰트 크기, 디바이스 선택, 창 크기·위치를 세션 간 저장·복원합니다.
- 메인 창 닫힐 때 모든 미리보기 창이 함께 닫힙니다.
- 8개 UI 언어 완전 지원

### 설정 패널 탭 내비게이션: 너비 조절 · 아이콘 · 라벨화

설정 패널의 사이드 내비게이션 영역 오른쪽 가장자리를 드래그하여 너비를 조절할 수 있으며, 각 탭에 아이콘과 라벨이 함께 표시됩니다.

- 드래그 범위: 최소 120px ~ 최대 300px, 조절한 너비는 자동으로 저장됩니다.
- 패널 너비가 현재 폰트 크기 기준으로 가장 긴 라벨에 맞춰 자동 설정되어 8개 언어 모두에서 올바르게 표시됩니다.
- 편집기·외관·자동 백업·휴지통·정보 탭에 아이콘 5종이 추가되었습니다.
- **정보** 탭이 내비게이션 패널 하단에 구분선과 함께 고정되어 앱 왼쪽 사이드바의 풋터 레이아웃과 일치합니다.

## 버그 수정

### 이미지·PDF 뷰어 확대 시 가로 스크롤(패닝) 미동작

창 너비를 넘어 확대했을 때 콘텐츠의 왼쪽 영역을 스크롤하여 볼 수 없었습니다. 수정되었습니다.

### 글꼴 크기 변경 시 편집기 하단 여백 오류

글꼴 크기 또는 줄간격 변경 후 에디터 하단에서 마지막 몇 줄이 잘려 보이는 경우가 있었습니다. 수정되었습니다.

### 미리보기 창 테마 실시간 미동기화

설정 화면이 열려 있는 상태에서 테마를 변경해도 미리보기 창에 즉시 반영되지 않았습니다. 수정되었습니다.

### 편집기 옵션 자동저장 토글 버튼 표시 오류

편집기 옵션 패널의 자동저장 토글 버튼 손잡이가 트랙 밖으로 표시되는 문제가 있었습니다. 수정되었습니다.

### Windows: 에디터에서 이미지 미표시

Windows에서 문서에 삽입된 이미지가 에디터에서 표시되지 않았습니다. 수정되었습니다.

### 다국어 설정 시 미리보기 창 제목 한글 고정

한국어 외 언어로 앱을 사용할 때 미리보기 창 제목이 한국어로 표시되었습니다. 수정되었습니다.

### 사이드바 너비 축소 시 배지 테두리 미제거

파일명이 잘릴 만큼 사이드바를 좁혀도 확장자 배지가 테두리와 함께 남아 있었습니다. 이제 해당 상황에서 배지가 숨겨집니다.

### 웹소설 모드에서 세계관·배경 폴더 붙여넣기 불가 버그

웹소설 모드 활성 시 세계관·배경 폴더에 파일을 붙여넣을 수 없었습니다. 수정되었습니다.

### Windows: 드래그앤드롭 시 워크스페이스 이름에 전체 경로 표시

Windows에서 폴더를 앱으로 드래그하면 폴더 이름 대신 전체 파일 경로가 워크스페이스 이름으로 표시되었습니다. 수정되었습니다.

### Windows: 컨텍스트 메뉴 "파일 탐색기에 표기" 표기 오류

Windows에서 일부 언어의 "파일 탐색기에 표기" 컨텍스트 메뉴 항목이 올바르게 표시되지 않았습니다. 수정되었습니다.

### 사이드바 인라인 파일명 변경 버그 3종 수정

인라인 이름 변경 흐름의 버그 3개가 수정되었습니다.

1. **이름 변경 시 미저장 내용 소실**: 미저장 내용이 있는 파일의 이름을 변경하면 내용이 사라졌습니다. 이제 이름 변경 전에 자동으로 저장됩니다.
2. **중복 파일명 시 묵시적 덮어쓰기**: 기존 파일과 같은 이름으로 변경하면 원본 파일이 조용히 덮어써졌습니다. 이제 충돌 시 입력창에 빨간 테두리가 표시되고 고유한 이름을 입력할 때까지 변경이 차단됩니다.
3. **잘못된 문자 허용**: `.`으로 시작하거나 `/`, `\`를 포함한 이름이 허용되었습니다. 이제 중복 충돌과 동일하게 빨간 테두리로 차단됩니다.

## 개선 사항

- **사이드바 리사이즈 핸들** 히트 영역이 넓어져 시각적 변화 없이 핸들을 쉽게 잡을 수 있습니다. 왼쪽 사이드바와 오른쪽 보조 사이드바 모두 적용되며, 4가지 테마에서 동일하게 동작합니다.
- **설정 패널 헤더** 하단 구분선이 UI 텍스트 크기 전 범위에서 사이드바 헤더와 동일한 높이에 정렬됩니다.
- **사이드바 하단 "설정" 버튼의 아이콘**이 UI 텍스트 크기 변경에 비례해 함께 스케일됩니다.
- **UI 텍스트 크기** 슬라이더 상한이 16px에서 24px로 확장되었습니다. 사이드바 헤더 라벨·액션 버튼 아이콘, 워크스페이스 이름 라벨·웹소설 모드 배지, 집필 모드 토글 버튼·아이콘, 범위 폴더·에피소드 제목 입력창, 웹소설 안내 패널 라벨·설명·버튼, 접힌 폴더 아이콘·호버 액션 아이콘이 새로 연동됩니다.
- 파일 또는 폴더를 **생성·이름 변경**할 때 허용되지 않는 이름(Windows 예약 장치명, `.`으로 시작, `/`·`\` 포함, 마침표·공백으로 끝나는 이름, 과길이)은 실행 전에 빨간 테두리와 인라인 안내 메시지로 차단됩니다. 8개 언어 모두 지원합니다.
